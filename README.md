1. 项目背景与目标
1.1 背景
•	消费者与企业采购方在购买商品时，难以穿透多层销售渠道直接了解真实生产厂家，供应链透明度低。
•	电商平台（如1688、淘宝）大量商品标注的“发货地”与实际生产地、生产厂家之间缺乏有效关联。
•	现有工具大多仅提供简单的发货地展示，未提供基于大模型推理的深度产地溯源报告。
1.2 目标
•	建设一个小程序化商品溯源系统，自动对接主流电商平台，通过识别货物发货地，结合企业工商信息、产业带知识图谱和大模型推理，溯源至最终生产厂。
•	为用户提供可信、直观、可解释的产地溯源报告，提升消费决策与供应链管理水平。
•	在合规合法的前提下获取数据，构建可持续运营的SaaS+增值服务商业模式。

2. 产品概述
•	产品名称：源瞳产地雷达（英文：OriginRadar）
•	产品形态：以微信小程序为主，支持H5、App多端发布。
•	用户群体：终端消费者、电商采购商、品牌方打假部门、供应链管理者。
•	核心价值：从“发货地”一键溯源“生产厂家”，并提供AI推理报告。

3. 用户角色
角色	描述
普通消费者	扫码或粘贴链接查询所购商品的真实产地，获取简易溯源报告。
采购商/卖家	批量查询供应商发货地是否与实际生产一致，调查潜在源头工厂。
品牌打假人员	监控疑似假冒产品发货链路，获取厂家关系图谱，用于维权取证。
系统管理员	管理数据源、监控API配额、审核异议申诉、配置大模型提示词。

4. 功能需求
4.1 信息获取模块
•	商品链接解析：输入1688、淘宝等平台商品详情页链接，自动获取商品标题、店铺名称、标注发货地等信息（优先使用开放平台API）。
•	快递面单OCR识别：用户拍照上传快递面单，系统调用OCR接口识别并提取“寄件地址”字段。
•	手动输入发货地：提供表单，用户手动填写已知发货地文本，系统进行处理。
4.2 地址标准化模块
•	对接地址解析服务，将任意格式的发货地文本转为结构化省/市/区数据。
•	若地址模糊，基于地理编码（逆地理解析）补全坐标和行政区划。
4.3 产业带匹配模块
•	内置中国重点产业带知识库，支持按省/市/区匹配当地典型产业（如：杭州萧山-羽绒服、深圳华强北-电子产品）。
•	对查询商品类目进行自动或手动匹配，输出发货地与产业带特征的一致程度评分。
4.4 企业关联与知识图谱模块
•	利用第三方企业信息API（天眼查/爱企查等），根据发货地坐标和地址，查询周边生产企业。
•	构建企业知识图谱（图数据库），存储企业注册地址、经营范围、投资关系、人员关联。
•	支持从发货店铺的工商主体出发，发现其与生产厂家的多级关联关系。
4.5 溯源推理与大模型报告生成模块
•	整合：标准化发货地、产业带匹配度、企业周边检索结果、经营范围、关联关系等信息。
•	通过大语言模型API生成结构化溯源推理报告，包含：
o	最可能的生产厂家（列表）
o	每个厂家的置信度评分
o	推理依据（地理位置、产业带、经营范围、关联关系、历史数据）
o	风险标记（经营异常、非生产型企业等）
4.6 可视化展示模块
•	发货轨迹地图：若有完整物流信息，展示发货路径；否则展示发货地点点。
•	企业关联图谱：力导向图展示店铺主体、中间人、生产厂家之间的股权/人员关系。
•	报告卡片：简洁版溯源报告，支持分享、下载PDF。
4.7 用户反馈与申诉
•	厂家可提交证明资料申诉溯源结果，管理员审核后可修正知识库。
•	用户可对溯源结果进行“有用/无用”评价，用于模型优化。

5. 数据需求
5.1 数据来源
数据类型	来源	获取方式
商品信息、发货地	电商平台（淘宝、1688）	开放平台API、合法爬虫（遵守Robots协议）
快递面单地址	用户上传图片	OCR识别接口
企业工商信息	天眼查/爱企查/企查查等	付费/免费API，按需缓存
产业带数据	自建/公开数据采集	定期维护更新
地理与地址解析	百度/腾讯地图API	免费额度支撑初期
物流轨迹	快递100/快递鸟等	用户授权后调用，非必选
大模型推理能力	GPT-4/文心一言等	API调用，按Token计费
5.2 数据存储
•	关系型数据库（MySQL）：用户信息、查询记录、任务队列、配置数据。
•	图数据库（Neo4j）：企业节点（店铺主体、生产厂家），关系边（投资、控股、人员、地址相近），用于多级关联分析。
•	缓存（Redis）：高频查询的企业信息、地址解析结果、产业带匹配结果，减轻API压力和成本。
5.3 数据清洗要求
•	去除地址中的特殊字符和无关信息。
•	统一行政区划名称（如“杭州萧山”标准化为“浙江省-杭州市-萧山区”）。
•	企业名称清洗（剔除个人店、无资质个体工商户等非生产性实体）。

6. 技术架构
6.1 前端
•	框架：uni-app (Vue.js)，实现一套代码发布微信小程序、H5。
•	组件：ECharts（地图/关系图）、Vant UI（移动端组件）。
•	核心能力：调起相机拍照OCR、定位授权、链接解析输入。
6.2 后端
•	业务服务：Python (FastAPI/Django) 或 Java (Spring Boot)，承载API和业务编排。
•	任务引擎：异步处理耗时操作（OCR识别、大模型调用），使用消息队列（Redis/Celery）。
•	定时任务：定期更新产业带数据、清理缓存、同步企业信息变更。
6.3 第三方服务集成
•	电商开放平台SDK：淘宝开放平台 TOP、1688开放平台。
•	OCR识别：阿里云/百度AI开放平台。
•	地址解析/地图：百度地图API 或 腾讯位置服务。
•	企业信息查询：天眼查/爱企查API（申请商务合作）。
•	大模型API：OpenAI API / 百度文心API，配置代理和降级策略。
6.4 部署与运维
•	初期：云服务器（4核8G）+ 云数据库，或直接使用云函数（Serverless）降低成本。
•	监控：Prometheus+Grafana 监控API调用量与延迟，Sentry收集异常。
•	容灾：对第三方API调用实现指数退避重试、熔断降级（返回简单发货地信息而不给出深度报告）。

7. 核心业务流程
1.	用户输入：
o	粘贴商品链接 / 拍照上传面单 / 手动输入发货地。
2.	数据获取与清洗：
o	若链接，调用电商API获取商品标题、店铺、发货地。
o	若图片，OCR提取地址文本；若手动，直接进入标准化。
o	地址解析为结构化省市区。
3.	特征分析：
o	查询产业带库，输出产业匹配度分数。
o	调用地图API，获取发货地周边（如半径50公里）的企业工商列表。
4.	知识图谱查询：
o	在图谱中搜索发货店铺主体的关联企业，重点关注“生产制造”经营范围的实体。
o	输出候选生产厂家集合。
5.	综合评分与排序：
o	基于地理位置距离、产业匹配、经营范围相符度、关联关系权重、企业规模等维度打分。
6.	大模型推理报告生成：
o	整合全部维度数据，发送至大模型，生成自然语言溯源报告。
o	返回结果中包含置信度和推理链条。
7.	结果展示：
o	地图标记发货地、厂家位置。
o	关系图谱（如有）。
o	报告文本摘要及详细信息。
流程图（文本描述）：
用户输入 → 数据采集（API/OCR/手动） → 地址解析 → 产业带匹配 + 企业周边搜索 + 图谱关联 → 多维度打分排序 → LLM生成报告 → 可视化呈现

8. 非功能需求
8.1 性能
•	普通溯源查询响应时间 ≤ 3秒（不含大模型深度报告）。
•	包含大模型报告的查询，平均响应 ≤ 15秒，超时则返回“简易报告+异步生成通知”。
•	系统支持初期50 QPS，可水平扩展。
8.2 安全
•	用户上传的图片、物流信息加密存储，7天后自动删除。
•	个人敏感信息（如姓名、电话）在OCR识别后自动脱敏。
•	API调用需携带令牌，防止伪造请求。
8.3 合规（重点）
•	数据来源：严格使用电商平台开放API，不进行非法爬取；不破解、绕过技术保护措施。
•	Robots协议：如需抓取公开信息，必须遵守目标网站的robots.txt，并严格控制频率。
•	用户授权：上传面单、获取物流轨迹前，明示隐私条款并取得同意。
•	企业信息使用：遵循第三方API服务协议，不缓存超量数据，不用于骚扰电话等。
•	大模型内容安全：对生成的报告内容进行过滤，避免涉及虚假宣传、侵犯商业信誉。
8.4 可维护性
•	第三方服务通过统一适配器层调用，方便替换。
•	大模型提示词采用配置化管理，可灵活调整报告风格。
•	产业带库提供后台管理界面，支持人工增删改查。

9. 运营与盈利模式
9.1 免费与付费
•	免费额度：每日3次基础溯源（仅显示发货地、初步产业带提示）。
•	付费订阅：
o	个人版：29.9元/月，每日20次深度报告。
o	专业版：99.9元/月，每日100次深度报告+图谱展示+历史记录导出。
o	企业版：2999元/月，API接口调用、批量溯源、定制品牌保护模块。
•	单次付费：深度溯源报告2.99元/次。
9.2 增值服务
•	供应链验厂报告：整合更多工商信息和大模型深度分析，收费99元/份。
•	品牌打假代运营：按维权效果抽佣10%~30%。
•	广告：供应链服务商广告位，CPT模式刊例价2000~5000元/月。
9.3 初期成本控制
•	使用免费额度的地图API、企业信息API基础版。
•	小程序采用uni-app开发，减少多端开发成本。
•	后端初期采用单机云服务器，按需扩展。

10. 风险与对策
风险	应对措施
电商平台API限制或收费	引导用户使用面单OCR作为补充；与平台洽谈合作获取授权。
溯源准确度不足	引入用户反馈机制，持续训练大模型提示词；人工审核热点商品数据。
法律合规风险	聘请法律顾问审查数据获取和报告生成流程；购买数据合规类保险。
第三方API成本过高	建立多层缓存机制，对高频企业信息采用自建数据库同步更新。
竞争模仿	快速积累产业带知识图谱和用户标注数据，形成数据壁垒；申请软件著作权和专利（溯源推理方法）。

11. 项目里程碑建议
•	Phase 1（MVP）：链接解析+OCR+地址标准化+产业带匹配+简单厂家搜索，输出基础报告。
•	Phase 2（核心）：知识图谱构建、大模型推理报告、可视化展示。
•	Phase 3（商业化）：付费订阅、增值报告、打假代运营。
•	Phase 4（生态）：开放给第三方开发者，提供溯源API；对接更多电商平台。

12. 附录：术语表
•	OCR：光学字符识别，用于识别图片中的文字。
•	知识图谱：用图结构存储实体（公司、人）及其关系，便于多级关联分析。
•	LLM：大语言模型，用于生成自然语言推理报告。
•	产业带：在特定区域形成的产业集聚区，如“虎门女装”、“永康五金”。
•	置信度：系统对溯源结果准确性的自我评估分数。
